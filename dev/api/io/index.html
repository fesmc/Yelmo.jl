Input / output API

The output module is in Yelmo.YelmoIO; restart-loading helpers live in Yelmo.YelmoCore and are documented on the core API page. Both surfaces dispatch on AbstractYelmoModel so YelmoModel and YelmoMirror share the same output API.

See the output guide for the narrative.

Output lifecycle

Yelmo.YelmoIO.init_output — Function
init_output(ylmo, path; selection, deflate, include_scratch) -> YelmoOutput

Create a NetCDF file at path. Defines all dimensions, coordinate variables, and one NetCDF variable per field that passes selection and has a recognised grid. Returns a YelmoOutput to be passed to write_output!.

Set include_scratch=true to also serialize solver-internal scratch buffers (e.g. dyn.scratch.sia_tau_xz) — useful for debugging the SIA solver's intermediate state. Off by default so production runs don't bloat NetCDFs with non-state fields. Scratch fields belong to their parent group, so OutputSelection's group filter still applies; their NetCDF variable names are prefixed with <group>_scratch_ (e.g. dyn_scratch_sia_tau_xz).

source
Yelmo.YelmoIO.write_output! — Function
write_output!(out::YelmoOutput, ylmo::AbstractYelmoModel; include_scratch)

Append one time slice (at ylmo.time) to the open output file.

include_scratch defaults to whatever init_output set up — i.e. true iff scratch entries were registered. Passing include_scratch=false explicitly suppresses scratch writes for a particular slice (the NetCDF variables remain, just unwritten at that time index).

source

Closing the file is via the standard close(out) — Base.close is extended for the internal YelmoOutput struct returned by init_output.

Variable selection

Yelmo.YelmoIO.OutputSelection — Type
OutputSelection(group_vars)

Select specific variables per group, e.g.:

OutputSelection([
    :tpo => ["H_ice", "z_srf"],
    :dyn => ["ux", "uy", r".*_acx$"],
])
source

Patterns are matched against variable names via the matches_patterns helper (a thin any(p -> occursin(p, name), …) wrapper). Patterns can be plain strings (substring match), regex literals, or any type accepted by occursin(p, name::String).

Variable metadata

The variable tables under src/variables/{model,mirror}/ are parsed into VariableMeta entries by parse_variable_table at constructor time. Field allocation, NetCDF dimension names, and per- group disambiguation all key off these entries.

Yelmo.YelmoMeta.parse_variable_table — Function
parse_variable_table(filename) -> NamedTuple{names, NTuple{N,VariableMeta}}

Parse a Markdown variable table into a NamedTuple of VariableMeta entries keyed by variable name (as Symbols). Expected columns: id | variable | dimensions | units | long_name.

Trough F17 — SSA / DIVA lockstep

Test files: test/benchmarks/test_trough.jl, test/benchmarks/test_trough_diva.jl Fixture: test/benchmarks/fixtures/trough_f17_t1000.nc Solvers tested: SSA, DIVA Validation: YelmoMirror reference fixture at t = 1000 yr

Setup

The Trough-F17 benchmark is an idealised marine-ice-stream geometry inspired by Feldmann et al. (2014) and Bueler & Brown (2009). A sinusoidal trough channels ice toward the ocean; the domain wraps in the y-direction so the trough is periodic.

ParameterValue
Grid88 × 21, dx = 8 km, Bounded x / Periodic y
Iceρ = 910 kg m³
Forcingsmb_ref = 0.5 m/yr, T_srf = 263.15 K, Q_geo = 42 mW m⁻²
Frictionbeta_method = 2 (Weertman power law), q = 1/3, u₀ = 31556926 m/yr
Tillmethod = 1, angle-based (is_angle = true), cf_ref = 10°, z0 = −300 m, z1 = 200 m
N_effmethod = −1 (loaded from fixture; subgrid sampling deferred)

The fixture is generated by YelmoMirror running the Trough F17 Fortran namelist from a cold zero-ice start to t = 1000 yr.

What it tests

test_trough.jl — round-trip + SSA dyn_step

  1. File-based load — reads trough_f17_t1000.nc with boundaries = :periodic_y; verifies shape and physical sanity (ice has grown, trough depth matches F17 geometry, ux_b > 50 m/yr on the grounded trunk).

  2. In-memory load — builds a YelmoModel via state(b, 1000.0), which reads the same fixture through the AbstractBenchmark interface. Center-aligned fields (H_ice, z_bed, f_grnd, T_srf) must agree to machine precision between the two load paths.

  3. SSA dyn_step! lockstep — runs one dyn_step! on the file-loaded model (which carries the full ATT / ux_b / cb_ref state from the fixture) and logs the relative error in ux_bar / uy_bar against the YelmoMirror reference. Picard convergence is asserted; the lockstep error is informational (to be tightened in follow-up).

test_trough_diva.jl — DIVA solver

Same fixture + DIVA (solver = "diva") in place of SSA-only. Exercises the hybrid DIVA kernel (SIA + SSA coupling) and validates that the DIVA solution is in the same neighbourhood as the SSA reference from the fixture.

How to run

julia --project=test test/benchmarks/test_trough.jl
julia --project=test test/benchmarks/test_trough_diva.jl

Both tests require the committed fixture fixtures/trough_f17_t1000.nc. To regenerate it from a local Fortran build:

julia --project=test test/benchmarks/regenerate.jl trough_f17 --overwrite

Benchmark struct

include("test/benchmarks/helpers.jl")
using .YelmoBenchmarks

b = TroughBenchmark(:F17; dx_km = 8.0)
y = YelmoModel(b, 1000.0; p=p, boundaries=:periodic_y)
# Or load from file directly:
y = YelmoModel("fixtures/trough_f17_t1000.nc", 1000.0; boundaries=:periodic_y, ...)

Notes on N_eff

The trough namelist uses yneff.method = 3 with nxi = 5 subgrid sampling, but subgrid sampling is deferred in Yelmo.jl. The lockstep test therefore uses yneff.method = −1 (external) so the fixture's N_eff field is loaded as-is, preserving agreement with the Fortran reference. Switching to method = 3 with nxi = 0 would compute a slightly different N_eff than Fortran's nxi = 5 result.

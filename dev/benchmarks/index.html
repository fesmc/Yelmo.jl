Benchmarks

The test/benchmarks/ directory holds end-to-end validation benchmarks for YelmoModel. Each benchmark exercises a specific solver or pipeline component against either a closed-form analytical solution or a committed YelmoMirror reference fixture generated by the Fortran backend.

Fixtures under test/benchmarks/fixtures/ are committed to the repository. CI loads them directly; test/benchmarks/regenerate.jl rebuilds them locally when you need to refresh after a physics change.

How to run

All benchmark test files activate the test/ project and are runnable as standalone scripts:

julia --project=test test/benchmarks/test_sia.jl
julia --project=test test/benchmarks/test_eismint_moving.jl
julia --project=test test/benchmarks/test_hom_c.jl
julia --project=test test/benchmarks/test_mismip3d_stnd.jl
julia --project=test test/benchmarks/test_mismip3d_stnd_lockstep.jl
julia --project=test test/benchmarks/test_mismip3d_stnd_att.jl
julia --project=test test/benchmarks/test_mismip3d_stnd_implicit.jl
julia --project=test test/benchmarks/test_trough.jl
julia --project=test test/benchmarks/test_trough_diva.jl
julia --project=test test/benchmarks/test_calvingmip_exp1.jl
julia --project=test test/benchmarks/test_slab_advection.jl
julia --project=test test/benchmarks/test_adaptive_dt.jl

The initMIP real-domain benchmarks (Greenland, Antarctica) each use their own project under benchmarks/initmip-grl/ and benchmarks/initmip-ant/ (separate from the test/ CI tree). The configuration is a native YelmoParameters built in run.jl (build_params); both backends are driven from the same parameters via the keyword main entry point:

cd benchmarks/initmip-grl                                          # or initmip-ant
julia --project=. -e 'include("run.jl"); main()'                   # YelmoModel
julia --project=. -e 'include("run.jl"); main(backend=:mirror)'    # YelmoMirror

The EISMINT-1 lockstep test, trough, and CalvingMIP regression tests require their committed YelmoMirror fixtures to be present in test/benchmarks/fixtures/. These are checked in and do not require a Fortran build to run against. To regenerate a fixture from scratch (requires a compiled libyelmo_c_api.so):

julia --project=test test/benchmarks/regenerate.jl <name> --overwrite
# e.g.: julia --project=test test/benchmarks/regenerate.jl trough_f17 --overwrite

Benchmark infrastructure

Two backend patterns are used depending on whether a closed-form solution exists:

AbstractBenchmark (analytical) — a struct carrying the benchmark parameters implements state(b, t) (returning a NamedTuple of schema-keyed fields) and analytical_velocity(b, t). A generic YelmoModel(b, t; ...) constructor builds a model directly from the analytical state with no NetCDF round-trip. Used by BUELER-B, EISMINT-1 Moving, ISMIP-HOM-C, MISMIP3D, and CalvingMIP.

BenchmarkSpec (YelmoMirror-generated fixtures) — a BenchmarkSpec carries a Yelmo Fortran namelist, synthetic-grid axes, an end_time, and an setup_initial_state! callback. run_mirror_benchmark!(spec) drives YelmoMirror and writes a NetCDF restart at each output time. load_fixture constructs a YelmoModel from the resulting file. Used by EISMINT-1 Moving (lockstep fixture), Trough F17, and CalvingMIP Exp 1.

See test/benchmarks/README.md for the full infrastructure description and instructions for adding new benchmarks.
